# abuseipdb-ufw-reporter

A lightweight Bash script that monitors UFW firewall blocks and automatically reports suspicious IP addresses to AbuseIPDB.

## Features

- Real-time UFW firewall log monitoring using `journalctl`
- Automatic reporting of blocked IP addresses to AbuseIPDB
- Cloudflare IP whitelist auto-update
- Automatic filtering of private and local IP addresses
- In-memory tracking system to prevent duplicate reports
- Hostname identification in AbuseIPDB comments
- Lightweight Bash implementation
- Optional systemd service for background execution

## UFW Setup

This script requires UFW to be installed, enabled, and logging firewall events.

Install UFW:

```bash
sudo apt update
sudo apt install ufw
```

Enable UFW:

```bash
sudo ufw enable
```

Enable firewall logging:

```bash
sudo ufw logging high
```

Check UFW status:

```bash
sudo ufw status verbose
```

Example output:

```text
Status: active
Logging: on (high)
```

The script monitors `[UFW BLOCK]` events generated by UFW through `journalctl`.

## Quick Start

```bash
# Clone repository
git clone https://github.com/goncalopolido/abuseipdb-ufw-reporter

# Enter directory
cd abuseipdb-ufw-reporter

# Create environment file
cp .env.example .env

# Add your AbuseIPDB API key
nano .env

# Make script executable
chmod +x abuseipdb.sh

# Run manually
./abuseipdb.sh
```

## Systemd Service (Run in Background)

To run the script automatically in the background and restart it after reboots, create:

```bash
/etc/systemd/system/abuseipdb.service
```

with the following content:

```ini
[Unit]
Description=Report Unauthorized Connection Attempts To AbuseIPDB
After=network.target

[Service]
Type=simple
ExecStart=/bin/bash /path/to/abuseipdb-ufw-reporter/abuseipdb.sh
Restart=always
User=root

[Install]
WantedBy=multi-user.target
```

Replace `/path/to/abuseipdb-ufw-reporter` with the actual installation path.

Enable and start the service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable abuseipdb
sudo systemctl start abuseipdb
sudo systemctl status abuseipdb
```

View live logs:

```bash
journalctl -u abuseipdb -f
```

## Configuration

Create a `.env` file in the root directory:

```env
ABUSEIPDB_API_KEY=your_api_key_here
```

## How it works

1. Monitors UFW firewall events using `journalctl`
2. Detects `[UFW BLOCK]` entries
3. Extracts the source IP and blocked destination port
4. Ignores:
   - Cloudflare IP ranges
   - Private network addresses
   - Localhost addresses
5. Checks if the IP was already reported in the last 15 minutes
6. Sends the report to AbuseIPDB
7. Adds the IP to the temporary tracking list

The script only detects connections blocked by UFW. Allowed connections or traffic blocked by other firewalls will not be reported.

## AbuseIPDB API Limits

The AbuseIPDB API has daily request limits depending on the account type:

| Account Type | Daily Limit |
|---|---:|
| Free account | 1,000 reports/checks per day |
| Verified webmaster account | 3,000 requests per day |
| Supporter account | Up to 5,000 requests per day |

The API rate limit resets daily at 00:00 UTC.

AbuseIPDB also limits duplicate reports. The same IP address can only be reported once every 15 minutes.

This script already includes a 15-minute duplicate protection system using an in-memory tracking list.

## Increasing the API Limit

If you need more than the default 1,000 daily requests, you can increase your account limits by:

- Verifying your website as a webmaster to increase the limit up to 3,000 requests per day.
- Supporting AbuseIPDB to potentially increase the limit up to 5,000 requests per day.

More information about account upgrades and supporter status can be found through AbuseIPDB's official channels.
